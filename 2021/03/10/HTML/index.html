HTML

HTML理论知识

定义及作用

HTML：用来告知浏览器如何组织页面的标记语言，由一系列元素组成。

作用：

  • 使文本片段在文档有不同含义

    段落、列表、表格

  • 结构化文档

    头部、内容、脚部、导航

  • 嵌入多媒体

    图片、视频

元素

元素解析

  • 开始标签：元素的开始

  • 结束标签：元素的结束

嵌套元素

将元素放入其他元素中

所有元素必须正确开始和结束

分类

  • 块级元素

    • 内容前后换行

      嵌套在内敛元素中也不会受到影响

  • 内联元素

    • 内容前后不换行

空元素

不是所有元素都有开始标题、结束标签、内容

<img src="url">

属性

属性包含元素的额外信息

属性的格式：

  1. 空格（和元素或属性之间）
  2. 属性名=”属性值”

布尔属性

没有值的属性，实际上是该属性的取值只能是属性名

disable

文档解析

  1. <!DOCTYPE html> 文档类型声明
  2. <html></html> 根元素，包含整个页面
  3. <head></head> 包含你想要包含在页面但又不想显示的信息
  4. <meta charset="utf-8"> 页面的字符集
  5. <title></title> 页面标题，出现在浏览器标签
  6. <body></body> 页面的显示内容

文字处理

HTML的主要工作是编辑文本结构、文本内容，以便浏览器能正确的显示

为什么要结构化

  1. 以便用户阅读时快速定位内容
  2. SEO（搜索引擎会将标题内容作为关键字）
  3. 便于CSS、JavaScript定位元素

  2. 标题顺序要正确
  3. 每个页面标题层次最多3，否则多页面

超链接

使得文档可以链接到其他资源（包括其他文档），或该文档的指定部分，从而使互联网成为互联的网络。

跳转到HTML文档的特定部分

contacts.html

最佳实践

  1. 内容保持短小、不要带URL、”链接到/跳转”等噪音

  2. 同一网站资源使用相对URL

    绝对URL：浏览器会通过DNS查找ip，再请求

    相对URL：浏览器直接在本服务器请求

  3. 链接到非HTML资源时，留下提示

  4. 链接到下载资源时使用download属性提供默认文件

    download属性仅适用于同源URL

    同源：协议、主机、端口都相同的URL
